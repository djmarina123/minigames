# Internacionalização (i18n)

O app suporta **português**, **inglês** e **espanhol**. Novas features devem usar o sistema de tradução — não strings fixas na UI.

## Stack

| Item | Local |
|---|---|
| ARB (fonte) | `lib/l10n/app_pt.arb` (template), `app_en.arb`, `app_es.arb` |
| Código gerado | `lib/l10n/app_localizations.dart` (não editar à mão) |
| Config | `l10n.yaml` + `generate: true` em `pubspec.yaml` |
| Idioma persistido | `LocaleRepository` (`core/locale/`) · chave `app_locale` |
| Idiomas suportados | `AppLocales` — `pt`, `en`, `es` |
| Seletor na UI | `SettingsPanel` no Perfil |

## Primeira abertura

Sem preferência salva, `AppLocales.resolveInitial()` usa o `languageCode` do dispositivo. Se não for `pt`/`en`/`es`, cai em **português** (`AppLocales.defaultLocale`).

## Onde obter textos

| Contexto | Como |
|---|---|
| Widget com `BuildContext` | `AppLocalizations.of(context)` |
| Metadados de jogo, prep, ajuda | extension `HubL10n` em `core/l10n/l10n_extensions.dart` |
| Conquistas / missões | `l10n.localizedAchievement(def)` / `localizedMission(def)` |
| Jogo Flame (sem context) | `L10nScope.of` — instalado pelo `builder` do `MaterialApp` |
| Testes unitários fora de widget | `await L10nScope.installForTest()` no `setUpAll` |

```dart
// Widget
final l10n = AppLocalizations.of(context);
Text(l10n.navGames);

// Catálogo / prep / ranking
final meta = l10n.localizedMetadata(game.metadata);

// Flame (HUD, labels flutuantes)
caption: L10nScope.of.hudTime,
text: L10nScope.of.gameSnakeCrashed,
```

## Adicionar string nova

1. Adicionar chave em **`app_pt.arb`** (template) com valor em português.
2. Traduzir a **mesma chave** em `app_en.arb` e `app_es.arb`.
3. Placeholders ICU: `"coinsEarned": "+{amount} moedas"` + bloco `@coinsEarned` com tipos.
4. `flutter pub get` (regenera `app_localizations*.dart`).
5. Usar no código via `l10n.chave` ou `l10n.chave(param)` — **nunca** string literal na UI.

Parâmetros gerados são **posicionais** (`l10n.statDailyStreakDays(count)`), não nomeados.

## Jogo novo

1. Manter `GameMetadata` no `*_game.dart` com `id` estável; título/descrição no código podem ficar em PT como fallback interno.
2. Registrar strings em ARB com prefixo `game<Id>…` (ex.: `gameSnakeTitle`, `gameSnakeHowToPlay`, `gameSnakeScoring`).
3. Mapear em `HubL10n`: `gameTitle`, `gameDescription`, `gameHelp`, `prepGroupLabel`, `prepChoiceLabel`, `prepChoiceSubtitle`.
4. HUD e labels in-game: chaves `hud*` / `game*` + `L10nScope.of` no `render()` ou construtor do Flame.
5. Helpers em `*_config.dart` que retornam texto ao usuário: usar `L10nScope.of` (testes precisam de `installForTest`).

Exibir no hub/prep/ranking sempre via `localizedMetadata()` — não `game.metadata.title` cru.

## Conquista / missão nova

1. Entrada em `achievements_catalog.dart` ou `missions_catalog.dart` (id estável).
2. Chaves ARB: `achievement<id>Title` / `Desc` ou `mission<id>Title` / `Desc`.
3. Lookup em `HubL10n.achievementTitle` / `missionTitle` (e `*Description`).

## Novo idioma (futuro)

1. `lib/l10n/app_XX.arb` com todas as chaves.
2. `Locale('xx')` em `AppLocales.supported`.
3. Label em `HubL10n.languageLabel`.
4. `flutter pub get` + testes.

## Anti-padrões

- `Text('Jogar')`, `tooltip: '…'`, `SnackBar` com string fixa.
- Texto de economia em `economy_copy.dart` na UI (usar ARB; `EconomyCopy` legado).
- `GamePrepDefinition` com labels só em PT **sem** mapeamento em `HubL10n` (prep screen localiza por `optionKey` + valor).
- Esquecer `app_en.arb` / `app_es.arb` — CI não valida paridade; conferir manualmente.
- `L10nScope.of` em teste de config sem `installForTest`.

## Testes

- `setupTestRepositories()` fixa `app_locale: 'pt'` para textos previsíveis.
- Widget/golden: `buildTestApp` inclui `LocaleRepository` + delegates.
- Config/HUD helpers: `setUpAll(() async { await L10nScope.installForTest(); })`.
- Asserções de texto: preferir chave/idioma explícito ou PT via locale fixo.
